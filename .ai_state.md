- **⚠️ HISTORICAL — component-axis confound (Phase 7i, 2026-06-11), now RESOLVED above.** The silver validation (`obs_t_silver.py`) showed the empirical-cluster fallback agrees with the headword-join only **0.1%** on their overlap — because they measure DIFFERENT AXES: the join assigns component by LOCATION (headword typo → `headword`), the heuristic by edit TYPE (`typo` → `orthography`). So the inferred fallback (34% of corpus) filed type-axis values (orthography/encoding) into the location-axis component column. Evidence: orthography 38.3% of ALL labels but 12.8% derived; encoding 2.6% all / **0.0% derived**. On derived (location) labels the real typology is **sense 40.5% > headword 21.7% > orthography 12.8% > markup 11.8% > citation 10.0%** — NOT orthography-dominated. Consequence: the H1 "surface vs meaning" framing conflated sense-LOCATION with meaning-EDIT and must be restated. **Recommended fix (Phase 8)**: two orthogonal axes — LOCATION component (derived-only: headword/grammar/citation/sense/markup/crossref/meta; join-failures = `unattributed`) × EDIT-TYPE (the existing ERRANT/OCR/textcrit crosswalk, where orthography/encoding/diacritic belong). This turns one muddled column into two clean axes (a stronger paper) but reworks attribution + all downstream stats/H1/page. Branch `obs-t-robustness` (Phases 7f campaigns, 7h robustness, 7i silver — committed, not pushed).
