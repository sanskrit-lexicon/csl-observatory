## 🧠 Dev Notes & Hypotheses

- **Boundary decision (2026-06-03)**: `csl-observatory` was narrowed to GitHub/org observability. The dictionary-content pipelines (L0 convention cladogram, forensic MW↔PWG lineage, lexico/R2 sense work, Article 21, sanhw1) were **moved to `../csl-atlas`** — confirmed present there with live history (PR #33). - **Data pipeline**: `scripts/pull_data.py` (GraphQL fetch) → `scripts/compute_metrics.py` (alias-merge onto canonical logins via `scripts/contributors_map.json`) → site CSVs in `observatory/site/src/data/`. `scripts/retry_via_clone.py` is the fallback for repos GraphQL can't serve (bare-clone `--filter=blob:none`, metadata only). `scripts/contributor_stats.py` is the **live-API** leaderboard; `scripts/bus_factor.py` is **offline/reproducible** over the committed CSVs.
- **Observable site** (`observatory/site/`, Framework v1): `npm run build` / `npm run dev` from that dir. Build output (`dist/`) and caches (`.observablehq/`, `src/.observablehq/`) are gitignored — commit only `src/*.md` and `src/data/*.csv`. Reactive `${}` values render as `<observablehq-loading>` placeholders in static HTML and hydrate client-side from FileAttachment data; a clean build is the compile-time signal.
- **Bus-factor headline** (current data): 65/76 repos have bus factor 1; funderburkjim alone = 51.8% of all contributions; core trio = 97.6%; Gini 0.86; 1/2/3 people cover 50/80/90% of work. Page math in `community.md` mirrors `scripts/bus_factor.py` so site and `reports/bus_factor.md` stay in sync.
- **Repo-health headline** (current data): 41/76 repos (53%) have no license at all, 21 are `NOASSERTION`, only 14 recognised SPDX; 46/76 default to `master`; 5 lack a description; 6 disposable `temp_*`/`test_*`/legacy repos are live; 0 stale (all pushed within ~3 weeks); only 5 fully clean. The org's risk is hygiene + concentration, not abandonment. `repo-health.md` site page mirrors `scripts/repo_health.py`.
- **Taxonomy-adoption headline** (current data): 5,324 issues (89 PRs excluded); 89% carry a type label, 83% severity, 79% milestone, **63% fully conformant** (1 type + 1 severity + milestone, per org CLAUDE.md). Adoption curve: 11–25% conformant 2014–18 → 92% peak 2025 → 39% dip 2026 (recent backlog, un-milestoned). 324 over-typed; 999 carry one of **54 distinct stray labels** (`Documentation` 195, `Correction submission` 175, `help wanted` 92, many `domain:*` — normalise/fold/delete candidates). Section on `/coverage` mirrors `scripts/taxonomy_adoption.py`.
- **Velocity-timeline headline** (current data): 9,877 commits / 5,324 issues opened over 2014–2026, but the busiest year (2021) drew only **11 distinct authors** — volume-per-person, not a growing base (reinforces bus-factor). PRs barely register: 89 in 13 years, 59% of them in 2026 (first PR 2019) — the org runs on direct commits + issues. Open-issue backlog peaked at 1,742 in 2025 (csl-orig correction wave) then fell to 913 in 2026 — correction-campaign signature. **NB**: `unique_authors` in `timeseries_annual/monthly.csv` is per-repo and must NOT be summed for an org total (double-counts); recompute distinct authors from `commits.csv` (author_login/date). `scripts/velocity_timeline.py` does this; the Activity-page authors chart was fixed to match.
- **Finding pattern (established, repeatable)**: offline script over a committed `src/data/*.csv` → `reports/<name>.md` + `src/data/<name>.csv` → Observable page/section (new page registered in `observablehq.config.js`, or a section added to an existing page), with `${}` KPI cards + Plot charts whose JS recomputes the same numbers as the script (cross-check + no drift). Four findings shipped this way: bus-factor (Community page), repo-health (new Repo Health page), taxonomy-adoption (Issue Taxonomy page), velocity-timeline (Activity page). **PowerShell gotcha**: a literal `&` in a `git commit -m @'...'@` here-string breaks parsing (git sees the message as pathspecs) — write "and" or use `git commit -F <file>`.
