# Baselines for the CDSL correction corpus (OBS-T)

_Generated by `scripts/obs_t_baselines.py` from `correction_events_release.csv`. Stdlib-only, deterministic; these are reference baselines that define the tasks, not tuned systems._

Three tasks the released resource supports, on a temporal split (train on the past, test on recent edits).

## 1. Error detection (form layer)

Minimal-pair ranking: does a character-trigram LM (trained on correct forms) assign higher likelihood to the corrected form than to the erroneous one? Chance = 0.5. An absolute-threshold classifier is uninformative because `old`/`new` differ by a single character.

| metric | value |
|---|---:|
| train tokens | 14,670 |
| test pairs | 3,127 |
| **pairwise accuracy** (chance 0.5) | **0.516** |
| tie rate | 0.012 |

## 2. Error correction (form layer)

Noisy-channel: pick the edit-1 candidate the character-LM scores highest (Norvig-style), preferring training-lexicon forms. Open-vocabulary, so it does not require the corrected form to have been seen before.

| metric | value |
|---|---:|
| train lexicon | 11,173 forms |
| test pairs | 3,127 |
| **accuracy@1** | **0.059** |
| share of test errors at edit-distance 1 | 0.787 (model's reach) |

## 3. Error-type classification (all derived events)

Predict the microstructure `error_component` from edit-op features (dominant operation, unit, edit-distance bucket, script, empirical cluster) with categorical Naive Bayes.

| metric | value |
|---|---:|
| train / test | 17,082 / 8,359 |
| classes | 7 |
| **accuracy** | **0.388** |
| macro-F1 | 0.302 |
| majority baseline (sense) | 0.226 |
