# Silver validation of the old fallback labels (OBS-T)

_Generated by `scripts/obs_t_silver.py`. A human-free reliability bound for the old empirical-cluster fallback: on form events where the csl-orig headword join succeeded (the more reliable signal), how often would the fallback heuristic have given the same component? The released table now keeps failed joins as `unattributed`; this report explains why._

| metric | value |
|---|---:|
| testable overlap (join-derived form events, heuristic non-abstaining) | 5,634 |
| **heuristic agreement with the join** | **0.1%** |
| inferred/unattributed rows in the corpus | 18,743 |

Where the two signals overlap they agree only **0.1%** of the time — and that near-zero number is a structural finding, not noise. The join assigns the component by **location** (a headword typo → `headword`), while the empirical heuristic assigns it by **edit type** (`typo` → `orthography`). They disagree because they measure **different axes**. The consequence: the inferred fallback was filing *type*-axis values (`orthography`, `encoding`) into the *location*-axis component column. On the trustworthy derived (location) labels the typology is led by **sense, headword, markup, citation** — not orthography, which is almost entirely a fallback artifact. **Release policy:** derive the microstructure (location) typology from derived labels only; keep edit-type (orthography/encoding/diacritic) in the ERRANT/OCR crosswalk where it belongs; label join-failures `unattributed` rather than guessing a component.
