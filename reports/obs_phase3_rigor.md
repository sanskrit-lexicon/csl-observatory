## H4 — correction labor is specialised beyond repo size

Login x family contingency (15 contributors x 11 families): chi2 = 1517.1, dof = 140, **Cramer's V = 0.131**. Contributor-level permutation test (shuffle the family-label pool across logins holding each login's total commit count fixed, 10,000 permutations — the repo-idiom substitute for an inferential p-value, since commits cluster by campaign): **permutation p = 0.0014**.

Descriptive companion — `normalized_entropy` across 17 logins: median **0.0**, bootstrap 95% CI [0.0, 0.8].

Null (independence / size-proportional allocation) is **rejected** at alpha=0.05. Labor concentrates on specific repos/families beyond what repo size alone would predict.

## H5 — edit-type profiles are corrector-invariant

Top-k correctors (>= 500 events): dhaval_ejf, drdhaval2785, funderburkjim (3 of 60 logins), restricted to rows with `edit_type != 'none'` (n = 48,542). Corrector x edit_type: chi2 = 27553.2, dof = 14; row-level p-values are not treated as inferential (events cluster by commit/campaign). Effect size: **Cramer's V = 0.533**, commit-block bootstrap 95% CI [0.526, 0.544] (the H2 idiom: resample commits, not events, since a single campaign commit can contribute thousands of same-type events).

The CI sits in a low-to-moderate V range, which is more mixed than the claim that edit-type mix reflects the material, not the person — but a non-zero V means correctors are not perfectly interchangeable.
