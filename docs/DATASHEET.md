# Datasheet — CDSL correction-event corpus (OBS-T)

Following Gebru et al., *Datasheets for Datasets* (2021). Describes the released
resource `observatory/site/src/data/correction_events_release.csv` and its
derivation. Snapshot 2026-06-11; regenerate with the OBS-T pipeline (below).

## Motivation

- **Purpose.** A twelve-year, multi-dictionary record of *corrections* to digitized
  Sanskrit dictionaries, typed by the microstructure component repaired and by
  edit operation, to support error typology, error detection/correction, and the
  study of digital-lexicography maintenance. Created for a computational-linguistics
  paper on the error profile of the Cologne Digital Sanskrit Lexicon (CDSL).
- **Created by.** M. Gasūns and the CDSL/sanskrit-lexicon community; assembled in
  `csl-observatory`. Source corrections by the CDSL correctors (named, alias-merged).

## Composition

- **Instances.** 50,953 correction *events*; one row = one old→new edit to a
  dictionary source, with date, dictionary, headword (IAST), old/new strings
  (IAST), edit-op trace, microstructure component, ERRANT/OCR/textual-criticism
  crosswalk labels, corrector, latency (form era), evidence label, and split.
- **Layers.** 24,441 from the 2014–2019 correction-form archive (`cfr.tsv`);
  26,512 reconstructed from the 2019–2026 `csl-orig` git history.
- **Coverage.** 43 dictionaries; 210 (alias-merged) correctors.
- **Labels.** `error_component` ∈ {headword, grammar, citation, sense, markup,
  crossref, meta, encoding, orthography, unknown}; each event carries an evidence
  label (`observed`/`derived`/`inferred`) — 65.7% derived, the rest heuristic.
- **Sampling.** Not a sample: the full available correction record. Known gaps —
  287 bulk-reformat git commits excluded; form-era events only ~28.6% machine-linked
  to current sources (encoding heterogeneity + L-code drift).
- **Sensitive data.** Corrector names (already public in the CDSL git history and
  `contributors_map.json`); raw email addresses are **not** included.

## Collection process

- **Form layer.** Exported from the Google "Sanskrit-Lexicon Correction form
  (Responses)" spreadsheet (`../CORRECTIONS/cfr.tsv`).
- **Git layer.** Mined from the sibling `../csl-orig` repository's commit history
  (correction-classified commits touching `v02/<dict>/<dict>.txt`).
- **Timeframe.** 2014-03-18 → 2026-05-30.

## Preprocessing / cleaning / labeling

- Normalized to **IAST** (NFC); the form's mixed Devanagari and **Harvard-Kyoto**
  encodings and the source's SLP1 are all transliterated by self-contained,
  stdlib-only maps. Inline commentary stripped from old/new cells.
- Edit operations computed over NFD (Damerau/OSA), typed by linguistic unit.
- **Component attribution:** git layer positionally from source XML tags; form
  layer by a fuzzy (edit-distance-1) headword join to `csl-orig`, else an
  empirical-cluster fallback.
- Raw cells preserved (`old_raw`/`new_raw`) for audit; nothing is discarded silently.

## Uses

- **Intended.** Error typology of digital dictionaries; Sanskrit error detection
  and correction; lexicographic maintenance and diachrony studies. Baselines in
  `reports/obs_t_baselines.md`.
- **Caveats.** Form-layer components are partly `inferred` (use the evidence label);
  form-era L-codes are drifted and should not be joined to current sources by id;
  the character-confusion matrix mixes IAST (form) and SLP1 (git) — filter by
  `layer` and `unit`.
- **Discouraged.** Treating `inferred` labels as ground truth; per-person
  performance judgements of correctors.

## Distribution

- **Where.** In `csl-observatory` (CC-BY-4.0); intended Zenodo deposit (DOI pending).
- **Splits.** Temporal `split` column: test = year ≥ 2025, dev = 2023–2024,
  train = earlier. Layer-specific baselines use a within-form chronological split.

## Maintenance

- **Maintainer.** `csl-observatory` maintainers.
- **Updates.** Regenerated by re-running the pipeline against fresher `csl-orig` /
  `cfr.tsv` snapshots; the snapshot date accompanies any citation.
- **Reproduce.**
  ```bash
  python scripts/build_correction_events.py      # form layer (cfr.tsv)
  python scripts/reconstruct_git_events.py       # git layer + merge
  python scripts/attribute_components.py          # microstructure component
  python scripts/attribute_crosswalks.py          # ERRANT/OCR/textcrit + confusion
  python scripts/obs_t_release.py                 # temporal split -> release csv
  python scripts/obs_t_typology.py                # stats + finding report
  python scripts/obs_t_baselines.py               # reference baselines
  ```
