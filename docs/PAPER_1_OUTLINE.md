# Observatory Report Outline

Date: 2026-06-04

Status: boundary-safe replacement for the older broad paper outline, preserved
as `PAPER_1_OUTLINE_LEGACY_BROAD_METRICS.md`.

This file is not a broad publication plan. It is a lightweight outline for a
possible report about the GitHub/org observatory itself.

## Scope

The report may explain how CDSL digitisation work appears in GitHub evidence:
repositories, issues, PRs, commits, contributors, workflows, labels,
milestones, repository metadata, and maintenance processes.

It must not analyze dictionary entries, dictionary structure, TEI/OntoLex
exports, DCS/corpus data, lookup telemetry, or publication programs outside
GitHub/org observability.

## Working Title

Measuring CDSL as a GitHub maintenance ecosystem

## Sections

1. Introduction
   - Why long-running digital lexicography needs reproducible maintenance
     metrics.
   - What this observatory measures and what it deliberately excludes.

2. Data And Reproducibility
   - GitHub API snapshots.
   - Repository metadata files.
   - Generated CSV/JSON chart data with snapshot dates.
   - Known caveats: permissions, rate limits, renamed users, and archived repos.

3. Repository Activity
   - Repos tracked by type.
   - Issues, PRs, commits, and releases over time.
   - Backlog and closure patterns.

4. Repository Health
   - License, citation, README, template, workflow, and release metadata.
   - Runbook adoption and documentation gaps.

5. Contributor And Maintainer Work
   - Contributor activity by year and repo.
   - Maintainer concentration and review load.
   - Identity-map limits and consent rules.

6. Organization Process
   - Cross-repo maintenance workflows.
   - Taxonomy coverage.
   - Artifact-refresh bottlenecks as repository/process evidence.

7. Discussion
   - What the observatory can support.
   - What it cannot claim.
   - Where dictionary, standards, corpus, and publication work now live.

## Out Of Scope For This Report

- Dictionary content or structure.
- TEI/OntoLex/FrAC standards work.
- DCS or any corpus evidence.
- Matomo/top-entry analytics.
- Wikipedia/Wiktionary backlink studies.
- Exact claims about labor value unless a separate methodology is approved.
