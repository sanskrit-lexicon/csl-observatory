# Activity timeline

Issue, PR, and commit throughput over 12 years across all 77 sanskrit-lexicon repos.

```js
const monthly = await FileAttachment("data/timeseries_monthly.csv").csv({typed: true});
const annual = await FileAttachment("data/timeseries_annual.csv").csv({typed: true});
const velocity = await FileAttachment("data/velocity_timeline.csv").csv({typed: true});
```

## Org-wide velocity & health

Organization-wide aggregates per year. Full finding: [`reports/velocity_timeline.md`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/reports/velocity_timeline.md) (generated by [`scripts/velocity_timeline.py`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/scripts/velocity_timeline.py)).

```js
const totCommits = d3.sum(velocity, d => d.commits);
const totIssues = d3.sum(velocity, d => d.issues_opened);
const totPRs = d3.sum(velocity, d => d.prs_opened);
const peakAuthors = d3.max(velocity, d => d.active_authors);
```

Throughput (commits, issues opened, issues closed) per year. The org runs on direct commits and issues — pull requests barely register until 2026.

```js
const flow = velocity.flatMap(d => [
  {year: d.year, metric: "commits", n: d.commits},
  {year: d.year, metric: "issues opened", n: d.issues_opened},
  {year: d.year, metric: "issues closed", n: d.issues_closed}
]);

Plot.plot({
  width,
  height: 360,
  marginRight: 100,
  x: {label: "Year", tickFormat: "d"},
  y: {label: "Count", grid: true},
  color: {legend: true, domain: ["commits", "issues opened", "issues closed"]},
  marks: [
    Plot.line(flow, {x: "year", y: "n", stroke: "metric", curve: "monotone-x", marker: "circle", tip: true}),
    Plot.ruleY([0])
  ]
})
```

Open-issue backlog — cumulative (issues opened − closed). It peaks in 2025 on the `csl-orig` correction wave, then falls sharply in 2026 as that wave is closed.

```js
Plot.plot({
```

## Activity heatmap: year × repo

```js
const repoSet = Array.from(new Set(annual.map(d => d.repo))).sort();
Plot.plot({
  width,
  height: 1200,
  marginLeft: 180,
  x: {label: "Year", domain: d3.range(2014, 2027)},
  y: {label: null, domain: repoSet},
  color: {scheme: "viridis", legend: true, label: "Issues+commits"},
  marks: [
    Plot.cell(annual, {
      x: d => +d.year,
      y: "repo",
      fill: d => d.issues_opened + d.commits,
      tip: true
    })
  ]
})
```

## Annual distinct commit-authors

Organization-wide distinct authors per year, recomputed from `commits.csv` (the per-repo `unique_authors` column cannot be summed — it double-counts people active in several repos). Even at peak, the active base is tiny — the concentration the [Community](/community) bus-factor analysis quantifies.

```js
Plot.plot({
  width,
  height: 300,
  x: {label: "Year", tickFormat: "d"},
  y: {label: "Distinct commit authors (org-wide)", grid: true},
  marks: [
    Plot.barY(velocity, {x: "year", y: "active_authors", fill: "#0075ca", tip: true}),
    Plot.ruleY([0])
  ]
})
```
