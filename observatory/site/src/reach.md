# External reach — CDSL's scholarly footprint

Who builds on, clones, ships, and cites the Cologne Digital Sanskrit Lexicon (CDSL) — framed as *scholarly reach*, not funder-facing vanity metrics (Workstream G6). **Measured** signals (stars/forks, GitHub traffic, downstream dependents) carry a URL and/or fetch date; the **estimated** citation tier is representative, with no completeness claim. Source: [`reports/external_reach.md`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/reports/external_reach.md), generated by [`scripts/external_reach.py`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/scripts/external_reach.py).

```js
const reach = await FileAttachment("data/external_reach.csv").csv({typed: true});
```

```js
const starsRow = reach.find(d => d.metric === "total_stars");
const forksRow = reach.find(d => d.metric === "total_forks");
const totalStars = starsRow ? starsRow.value : 0;
const totalForks = forksRow ? forksRow.value : 0;
const traffic = reach.filter(d => d.signal_type === "traffic");
const totalClones = d3.sum(traffic, d => d.value);
const consumers = reach.filter(d => d.signal_type === "dependent");
const codeDeps = reach.filter(d => d.signal_type === "dependent_codesearch");
const citations = reach.filter(d => d.signal_type === "citation");
const zenodo = reach.find(d => d.signal_type === "zenodo");
const fetchedAt = (traffic[0] || starsRow || {}).fetched_at;
```

The **stars-vs-clones gap is the finding.** The org collects roughly ${totalStars} stars in total, yet its core repositories are cloned **${totalClones.toLocaleString()} times in a single 14-day window**. CDSL is consumed as *infrastructure* — cloned, mirrored, and served through the Cologne website — not favourited on GitHub. Star count badly understates reach.

## GitHub traffic — the real usage signal

> **How to read:** Clones over a rolling 14-day window (GitHub only exposes two weeks, and only to accounts with push access). This is a **sample** of core infrastructure repos, not all 76, and the window slides — read it as a spot measurement, not a cumulative total. Bars are clone counts; the tooltip notes unique cloners. **Example:** `csl-orig` — the master dictionary source — draws the heaviest clone traffic because every downstream build pulls it.

```js
Plot.plot({
  width,
  height: 40 + traffic.length * 26,
  marginLeft: 130,
  x: {label: "Clones (14-day window)", grid: true},
  y: {label: null, domain: traffic.map(d => d.subject)},
  marks: [
    Plot.barX(traffic, {x: "value", y: "subject", fill: "#3a5f7d", tip: true,
      title: d => `${d.subject}: ${d.value} clones — ${d.note}`}),
    Plot.text(traffic, {x: "value", y: "subject", text: d => d.value.toLocaleString(),
      dx: 4, textAnchor: "start", fill: "black"}),
    Plot.ruleX([0])
  ]
})
```

> **Conclusion:** Two weeks of clones (${totalClones.toLocaleString()} across the sampled core) dwarf thirteen years of stars (${totalStars}). The audience is builders and mirrors pulling data programmatically, not GitHub stargazers — exactly the profile of a piece of scholarly infrastructure.

## Downstream dependents — who ships CDSL

Third-party projects that ship, wrap, or serve CDSL data. Each is a URL-checkable project that chose the Cologne lexicon as its lexical backbone — the strongest scholar-facing reach evidence.
