# Repository health

Repository-level hygiene across the org: licensing, default-branch naming, descriptions, and staleness. Object of analysis is the repository itself — not dictionary content. Source: [`reports/repo_health.md`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/reports/repo_health.md), generated by [`scripts/repo_health.py`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/scripts/repo_health.py).

Regression guard: [`scripts/repo_health_regression.py`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/scripts/repo_health_regression.py).

```js
const health = await FileAttachment("data/repo_health.csv").csv({typed: true});
```

```js
const n = health.length;
const licCounts = d3.rollup(health, v => v.length, d => d.license_class);
const noLicense = licCounts.get("none") ?? 0;
const unrecognised = licCounts.get("unrecognised") ?? 0;
const recognised = licCounts.get("recognised") ?? 0;
const legacyBranch = health.filter(d => d.default_branch === "master").length;
const cleanRepos = health.filter(d => d.flag_count === 0).length;
const hasFlag = (d, flag) => String(d.flags ?? "").split("|").includes(flag);
const cleanupRepos = health.filter(d => hasFlag(d, "cleanup-candidate"));
```

## Licensing

After the RH1 license rollout (2026-06), almost every repository carries a recognised SPDX license: code/tooling under GPL-3.0, dictionary and data repositories under CC-BY-SA-4.0, with mixed repos split (data CC-BY-SA-4.0 + `licenses/GPL-3.0.txt`). The `NOASSERTION` block is cleared. The only repositories still without a license are the RH3 archive/temp candidates, intentionally excluded until that cleanup runs.

```js
const licOrder = [
  {key: "none", label: "No license", color: "#cf222e"},
  {key: "unrecognised", label: "Unrecognised (NOASSERTION)", color: "#bf8700"},
  {key: "recognised", label: "Recognised SPDX", color: "#1a7f37"}
];
const licData = licOrder.map(d => ({label: d.label, count: licCounts.get(d.key) ?? 0, color: d.color}));

Plot.plot({
  width,
  height: 180,
  marginLeft: 200,
  x: {label: "Repositories", grid: true},
  y: {label: null, domain: licData.map(d => d.label)},
  marks: [
    Plot.barX(licData, {x: "count", y: "label", fill: "color", tip: true}),
    Plot.text(licData, {x: "count", y: "label", text: "count", dx: 12}),
    Plot.ruleX([0])
  ]
})
```

## License decision queue

The dashboard separates licensing into the same buckets as the maintainer
decision packet. **RH1 is complete (2026-06)** — the no-license and `NOASSERTION`
backlogs are cleared except the RH3-excluded archive/temp repos. See
[`docs/RH1_LICENSE_ROLLOUT_LOG.md`](https://github.com/sanskrit-lexicon/csl-observatory/blob/main/docs/RH1_LICENSE_ROLLOUT_LOG.md).

```js
const issueUrl = n => `https://github.com/sanskrit-lexicon/csl-observatory/issues/${n}`;
const decisionPacket = "https://github.com/sanskrit-lexicon/csl-observatory/blob/main/docs/REPOSITORY_HEALTH_DECISION_PACKET.md";
const repoUrl = repo => `https://github.com/sanskrit-lexicon/${repo}`;

display(html`<table>
```

```js
const licenseBacklogGroups = [
  ["No license", "none", 15],
  ["NOASSERTION", "unrecognised", 16],
  ["Recognised SPDX", "recognised", null]
];

function licenseRows(key) {
  return health
    .filter(d => d.license_class === key)
    .sort((a, b) => d3.ascending(a.repo, b.repo));
}

display(html`${licenseBacklogGroups.map(([label, key, issue], index) => {
  const rows = licenseRows(key);
  const detailBody = html`
    <summary><strong>${label}</strong> (${rows.length})${issue ? html` - tracker <a href=${issueUrl(issue)}>#${issue}</a>` : ""}</summary>
    <table>
      <thead><tr><th>Repository</th><th>Branch</th><th>Open issues</th><th>Flags</th></tr></thead>
      <tbody>
        ${rows.map(d => html`<tr>
          <td><a href=${repoUrl(d.repo)}>${d.repo}</a></td>
          <td><code>${d.default_branch}</code></td>
          <td>${d.open_issues}</td>
          <td>${d.flags || "clean"}</td>
        </tr>`)}
      </tbody>
    </table>
  `;
  return index < 2 ? html`<details open>${detailBody}</details>` : html`<details>${detailBody}</details>`;
})}`);
```

## Default branch

Branch naming is inconsistent — a mix of `master`, `main`, and `gh-pages` complicates tooling, CI defaults, and runbooks.

```js
const branchData = Array.from(
  d3.rollup(health, v => v.length, d => d.default_branch),
  ([branch, count]) => ({branch, count})
).sort((a, b) => b.count - a.count);

Plot.plot({
  width,
  height: 160,
  marginLeft: 100,
  x: {label: "Repositories", grid: true},
  y: {label: null, domain: branchData.map(d => d.branch)},
  marks: [
    Plot.barX(branchData, {x: "count", y: "branch", fill: d => d.branch === "main" ? "#1a7f37" : "#bf8700", tip: true}),
    Plot.text(branchData, {x: "count", y: "branch", text: "count", dx: 12}),
    Plot.ruleX([0])
  ]
})
```

## Cleanup decision queue

These repositories look temporary or legacy, but archiving stays blocked until
the maintainer confirms that the work was merged, superseded, or intentionally
preserved elsewhere.

```js
const cleanupNotes = new Map([
  ["santamlegacy", "Confirm no current deployment depends on the legacy PHP snapshot."],
  ["temp_corrections_acc", "Confirm ACC corrections were merged or superseded."],
  ["temp_corrections_ae", "Confirm Apte English-Sanskrit corrections were merged or superseded."],
  ["temp_corrections_ap90", "Review or migrate the open issue, then confirm the analysis repo is superseded."],
  ["temp_corrections_mw", "Review or migrate the open issue, then confirm the correction batch is superseded."],
  ["test_cologne_push", "Confirm the Cologne push test is no longer active."]
]);
const cleanupRows = cleanupRepos.slice().sort((a, b) => d3.ascending(a.repo, b.repo));

display(html`<table>
  <thead>
    <tr><th>Repository</th><th>Open issues</th><th>Days idle</th><th>Recommendation</th><th>Status</th><th>Tracker</th></tr>
  </thead>
  <tbody>
    ${cleanupRows.map(d => html`<tr>
      <td><a href=${repoUrl(d.repo)}>${d.repo}</a></td>
      <td>${d.open_issues}</td>
      <td>${d.days_since_push}</td>
      <td>${cleanupNotes.get(d.repo) ?? "Confirm merged/superseded state before archive."}</td>
      <td>blocked on RH3 confirmation</td>
      <td><a href=${issueUrl(19)}>#19</a></td>
    </tr>`)}
  </tbody>
</table>`);
```

## Repositories by flag count

Each repository's hygiene-flag count. Hover for the specific flags. Five repos are fully clean; the rest carry at least one gap.

```js
const ranked = health.slice().sort((a, b) => b.flag_count - a.flag_count || d3.ascending(a.repo, b.repo));

Plot.plot({
  width,
  height: 1100,
  marginLeft: 170,
  x: {label: "Hygiene flags", grid: true, domain: [0, d3.max(health, d => d.flag_count)]},
  y: {label: null, domain: ranked.map(d => d.repo)},
  color: {legend: true, domain: ["1+ flag", "clean"], range: ["#bf8700", "#1a7f37"]},
  marks: [
    Plot.barX(ranked, {
      x: "flag_count",
      y: "repo",
      fill: d => d.flag_count === 0 ? "clean" : "1+ flag",
      tip: true,
      channels: {License: "license", Branch: "default_branch", Flags: "flags"}
    }),
    Plot.ruleX([0])
  ]
})
```

Of **${n}** repositories, only **${cleanRepos}** are free of hygiene flags. The dominant gaps are licensing (**${noLicense}** with none, **${unrecognised}** unrecognised) and the legacy **${legacyBranch}** repos still defaulting to `master`. Staleness is not a problem — every repo was pushed within the last few weeks, so the ecosystem's risk is concentration and hygiene, not abandonment (see [Community](/community)).
